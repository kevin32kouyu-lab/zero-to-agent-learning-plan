### 第1天：ReAct模式原理

**🎯 目标**：理解ReAct，这是当前Agent最主流的工作范式

**上午（2h）核心知识点**
1. (1h) ReAct = **Reasoning + Acting**，核心是循环：
   > **思考 → 行动 → 观察 → 思考** ... 直到任务完成
   - **思考(Reasoning)**：LLM根据当前上下文分析，决定下一步做什么，要不要调用工具
   - **行动(Acting)**：执行决策，调用工具获取外部信息
   - **观察(Observation)**：把工具返回结果放回上下文，进入下一轮循环

   为什么好？把**推理**和**外部工具调用**结合起来，解决了LLM幻觉和知识截止问题。

2. (1h) 和你之前知道的CoT（思维链）对比：
   - CoT：LLM只思考，不行动，不能用外部工具
   - ReAct：思考+行动结合，能调用工具，能解决需要实时信息/外部数据的问题

**下午（2h）动手**
1. (2h) **不用任何框架，手写一个最简单的ReAct循环**，理解每一步做什么：

   简化代码骨架：
   ```python
   def react_agent(question, max_steps=5):
       context = [{"role": "user", "content": question}]
       for i in range(max_steps):
           # 1. 思考：LLM决定下一步要不要调用工具
           response = llm(context)
           # 2. 如果不需要调用工具，直接返回答案
           if is_finished(response):
               return get_answer(response)
           # 3. 需要调用工具：执行工具，拿到结果
           tool_name, tool_args = parse_tool_call(response)
           observation = run_tool(tool_name, tool_args)
           # 4. 观察：把结果放回上下文，继续循环
           context.append({"role": "tool", "content": observation})
       return "达到最大步数，未完成"
   ```
   手写一遍，你就真正理解ReAct了，不只是会用框架。

### 第2天：Agent记忆管理

**🎯 目标**：理解怎么让Agent记住东西，实现长期记忆

**上午（2h）核心知识点**
Agent的记忆分三层，各有作用：

1. **短期记忆**
   - 就是当前对话的上下文，直接放在LLM的上下文窗口里
   - 优点：快，直接能用；缺点：窗口有限，不能存太多

2. **长期记忆**
   - 历史对话/用户信息，转成Embedding存在向量数据库
   - 用户提问的时候，检索相关信息注入上下文
   - 优点：能存很多，不占窗口；缺点：多一步检索，有一点开销

3. **总结记忆**
   - 对长对话定期做摘要总结，保留核心信息，扔掉细节
   - 平衡信息完整性和窗口大小限制

**下午（2h）动手**
1. (2h) **给你之前的Agent加上简单长期记忆**：
   - 把每轮用户对话和Agent回复存在Chroma向量库
   - 用户新提问时，检索和问题相关的历史对话
   - 把检索到的历史放进Prompt，让Agent记得你之前说过什么
   测试：问Agent"我之前说过什么"，能正确回答就成功了

### 第3天：主流Agent框架对比

**🎯 目标**：知道每个框架适合什么场景，选对工具不瞎忙

**上午（2h）核心知识点**
| 框架 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| **LangGraph** | 自定义复杂Agent，复杂工作流编排 | 状态管理强大，灵活，生态好 | 需要写点代码，入门比低代码慢 |
| **CrewAI** | 多智能体协作 | 角色分工清晰，几分钟就能跑起来 | 复杂自定义流程不如LangGraph灵活 |
| **AutoGen** | 多智能体自由对话，支持人类介入 | 微软官方，成熟，功能强大 | 更适合对话场景，固定流程稍重 |
| **Dify** | 生产级低代码Agent，快速上线 | 可视化，RAG/Agent/发布一条龙，开发快 | 高度自定义不如手写框架 |

**选择原则**（记下来）：
- 快速原型/产品上线 → 选Dify
- 自定义复杂工作流 → 选LangGraph
- 多智能体协作 → 选CrewAI
- 多智能体自由对话 → 选AutoGen

**下午（2h）动手体验**
1. (2h) 每个框架都安装一下，跑通官方最简单的hello world，感受开发体验差异，选你喜欢的

### 第4天：LangGraph核心概念

**🎯 目标**：LangGraph是当前最流行的自定义Agent框架，理解核心设计

**上午（2h）核心知识点**
LangGraph基于LangChain，解决了Agent状态管理的问题，核心三个概念：

1. **节点(Node)**
   - 每个节点只做一件事：比如调用LLM、调用工具、输出结果
   - 你可以自定义任意节点，灵活组合

2. **边(Edge)**
   - 定义节点之间的流转关系
   - 支持**条件分支**：根据上一个节点的输出，决定下一个走哪个节点
   - 比如：LLM说需要调用工具 → 走工具节点；不需要 → 走输出节点

3. **State(状态)**
   - 整个Agent运行过程中，数据都存在State里，所有节点共享
   - 定义你的State结构，每个节点可以修改State

**下午（2h）动手**
1. (2h) 跑通官方第一个示例：简单聊天机器人，理解状态怎么流转
   - 官方入门：https://langchain-ai.github.io/langgraph/tutorials/introduction/
   跟着做一遍，就理解核心了
